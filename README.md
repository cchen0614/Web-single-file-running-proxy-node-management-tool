# 代理节点管理工具单文件运行版 v1.0  
  代理冲浪、上网爱好者，经常要对收集的节点，进行导入、导出，复制、粘贴，去重复等管理工作，但目前比较流行的代理客户端，对此支持都不十分友好，大多不支持批量导入、导出文本文件，或者是对节点管理有数量限制，比如，Winxray。或是作者默认的管理规则，不允许自定义“类型”、“别名”、"地址"、"端口"、"传输协议"、“TLS”去重复，比如v2ray。导致节点管理不尽人意。比如v2ray作者，对自定义去重复功能，不以为然，虽然使用者多次提出建议，但他感觉太小儿科，让使用者自己解决，不屑添加这个功能，等等。  
  本人根据这个日常需求，利用google的Ai studio编写了一个Web端的单文件运行版的代理节点管理工具，支持目前流行的Vmess、Vless、Trojan、ShadowSocks、ShadowSocksR等节点规则；在管理工具中，每个节点信息依次按“类型”、“别名”、"地址"、"端口"、"传输协议"、“TLS”这个通用格式排列；可以以节点源、txt文本文件、订阅链接形式把节点导入客户端，也可以以txt文本文件、复制到剪切板的形式，把整理好的节点再次导入客户端或导出；提供了“智能去重复”（自定义去重复）、“执行去重复”、“复制所选节点”、“导出txt文件”、“清空列表“等功能。  
”代理节点管理工具单文件运行版 v1.0“的开发过程。首先，代理节点管理工具“web端”是由google的Ai studio中的”Gemini 3 Pro Preview“模型开发的，只能在这个开发环境或服务器上，通过模块化加载（Import/Export）的方式运行。为了方便使用，我用这个模型又开发了 ”代理节点管理工具单文件运行版 v1.0“这个版本，完全离线运行（依赖 CDN，但无需本地服务器）。包含 JSZip，即使在单文件版里也能继续使用“下载源码”功能。如果您想修改单文件版的功能，正确的流程是：修改 ZIP 包里的源码 -> 重新生成/打包成单文件。这个单文件版 v1.0：就是将 Web 版的所有源代码文件、依赖库（React, Tailwind 等）进行“打包”和“转译”，合并到了一个独立的 .html 文件中。  
  如何使用：这个单文件运行版的使用和运行非常简单，将.html文件下载到本地，存储在任意路径。右键-“打开方式”-选择您使用的浏览器（比如Edge或Chrome）单击，或者“双击”，就可以在您的默认浏览器中打开，进入Web页面后，左侧是节点信息栏，按右侧Ui界面功能键操作即可。
